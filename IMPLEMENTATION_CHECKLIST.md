## Locked-in design constraints (from plan §2)

- `releaseStartMz?: boolean` and `releaseEndMz?: boolean` are **flat siblings on `Element`**, NOT nested under a `releases` object.
- Schema bump v1.0.0 → v1.1.0; migration registry adds an identity entry; v1.0.0 files load with rigid defaults.
- `2dfea/public/schemas/2dfea-model-v1.json` is regenerated by the `prebuild` script — committing the regenerated file is part of this feature.
- i/j labels appear on **element hover/select only** — never on node hover.
- Floating canvas panel buttons are **set / clear** semantics — never toggle. Four buttons: Release start / Release end / Release both / Clear releases.
- `def_releases(...)` in `pynite_analyzer.py` is called only when at least one of `releaseStartMz` / `releaseEndMz` is true.
- Both-ends-released is allowed (truss / simply-supported case); no JS-side stability check.
- Direct-push to master is blocked. PR with rebase-merge only; `pr-checks.yml` must go green.
